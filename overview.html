Overview#

MoonSim#

MoonSim is a suite of R program that predicts the ground illuminance of moonlight, twilight, and sunlight at a any geographic location and time period. The three MoonSim R programs generate different outputs:

  1. MoonSim: Lux calculator simply predicts moonlight and twilight illuminance.

    • For example, an ecologist could test for the effect of natural nocturnal light on bat activity level, by using the illuminance prediction as a predictor variable in a statistical model.

  2. MoonSim: Moonlight scheduler must be used with MoonShine (see below). It predicts only moonlight illuminance, and convert them into LED intensity values. Outputs a LED_schedule_moon.csv table that serves as a schedule for MoonShine.

  3. MoonSim: Sunlight & twilight scheduler must be used with MoonShine (see below). It predicts sunlight and twilight illuminance, and convert them into LED intensity values. Outputs a LED_schedule_sun.csv table that serves as a schedule for MoonShine.

MoonShine#

MoonShine is an LED controlling system for recreating realistic illuminance of moonlight, twilight, and to an extent â€“ sunlight in a laboratory. Moreover, the light schedule can be freely edited, making it ideal for experiments requiring a specific lighting scenario. MoonShine runs a python program on a Raspberry Pi to control an array of LED strips. Every minute, the python program reads the light schedule (generated by MoonSim: schedulers) to refresh the LED intensities. User can run a moonlight schedule and a sunlight and twilight schedule simultaneously on the Pi. As such, the Pi individually control two arrays of LED strips, to fully recreate all three major sources of natural light around the clock.
